## 🚀 Running Locally

### Prerequisites
- Python 3.9 or higher
- pip

### Step 1 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 2 — Run the analytics engine (generates outputs & figures)
```bash
python analysis_engine.py
```

### Step 3 — Launch the Streamlit dashboard
```bash
streamlit run app.py
```

The dashboard will open at `http://localhost:8501`

```bash

git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/finwise-ai-dashboard.git
git push -u origin main
```

### Step 2 — Deploy on Streamlit Cloud
1. Go to [share.streamlit.io](https://share.streamlit.io)
2. Click **"New app"**
3. Connect your GitHub repository
4. Set **Main file path** to `app.py`
5. Click **"Deploy"**

### Important Notes for Streamlit Cloud
- Ensure `data/` folder with all 5 CSV files is committed to the repo
- Ensure `outputs/` folder with `analysis_results.json` is committed
- Add `outputs/` pkl files OR set `analysis_engine.py` to run on first boot
- The `requirements.txt` is auto-detected by Streamlit Cloud

## 📝 Model Explanations

### Classification (WTP Binary)
- **Target**: Q20_WTP_Binary (0 = not willing, 1 = willing to pay)
- **Class balance**: 61.1% positive — no resampling required
- **Leakage prevention**: WTP intent, tier, and WTPperPain excluded from features
- **Best metric**: F1-score chosen over accuracy due to class imbalance

### Clustering (K-Means)
- **k=5** selected via Elbow + Silhouette analysis
- **Features**: 44 behavioural + attitudinal numeric variables
- **Validation**: Silhouette score = 0.080 (expected for high-dimensional survey data)
- **5 segments**: Budget-Conscious Expats, Digital-First Millennials, Finance Professionals,
  High-Value Nomads, Cautious Savers

### Association Mining (Apriori)
- **Input**: 34-column binary basket (multi-select survey responses)
- **Default thresholds**: support ≥ 0.10, confidence ≥ 0.55, lift ≥ 1.10
- **Output**: 1,388 meaningful rules after filtering

### Regression
- **Target 1 (WTP)**: R² ~0.30 — moderate, consistent with survey price studies
- **Target 2 (Download Likelihood)**: R² ~0.19 — intent is harder to model from behaviour
- **Key WTP drivers**: Income tier, pain composite, investment tracking preference, n_currencies

## ⚠️ Limitations

- Dataset is **synthetic** (1,200 simulated respondents) — real survey validation needed
- Stated WTP surveys typically **overestimate** actual conversion by 4–8×
- Silhouette score is low due to high feature dimensionality — PCA first may improve
- Regression R² moderate — reflects genuine uncertainty in stated-preference data
